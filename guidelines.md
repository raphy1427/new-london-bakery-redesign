# Codex Guidelines — Local Business Website Audit + Redesign Proposal System

## 1. Mission
Find small real-world businesses with outdated or low-quality websites, then generate **client-ready redesign proposals** that can be used for legitimate outreach.

Output is strictly for:
- portfolio demonstrations
- freelance outreach proposals
- redesign concept work with permission

No deception. No impersonation. No automated mass targeting.

---

## 2. Core Pipeline

### STEP 1 — Target Discovery (Search Phase)

Search the web for **small/local businesses** in categories:

- Restaurants
- Cafes
- Bakeries
- Gyms / fitness studios
- Barber shops / salons
- Auto repair shops
- Small retail stores
- Local service businesses

#### Search strategy rules:
Use structured queries like:
- “small restaurant + city”
- “family owned cafe + location”
- “barber shop near + city”
- “local bakery official website”
- “independent store + town”

Rotate location focus:
- small cities
- suburban areas
- tourist towns
- older business districts

Avoid:
- Fortune 500 companies
- modern tech startups
- chains with corporate websites

---

### STEP 2 — Website Validation

For each candidate:
- Open official website only
- Confirm it is actively representing the business
- Reject:
  - social media pages only
  - marketplace listings
  - directory pages (Yelp, Google Maps pages)

---

### STEP 3 — Design Quality Audit (Strict Scoring System)

Evaluate each website on a 20-point system:

#### 1. Visual Design (0–5)
- outdated layout
- poor spacing
- inconsistent fonts
- low-quality images

#### 2. Mobile Experience (0–5)
- broken mobile layout
- unreadable text
- poor responsiveness

#### 3. Navigation (0–5)
- confusing menu structure
- missing pages
- hard to find contact/info

#### 4. Performance Feel (0–5)
- slow loading
- heavy assets
- outdated scripts

RESULT RULE:
- ONLY select websites scoring **≤ 10/20**
- These are “high redesign value targets”

---

### STEP 4 — Selection Rule

Return only:
- 1 or 2 businesses per run

Prioritize:
- visibly outdated design
- clear usability issues
- high improvement potential

---

## 3. Output Format (STRICT TEMPLATE)

For each selected business:

---

### 1. BUSINESS PROFILE
- Name:
- Category:
- Location:
- Website:

---

### 2. CURRENT WEBSITE PROBLEMS

List clearly:
- Design issues
- UX/navigation issues
- Mobile issues
- Trust/credibility issues
- Missing features

No fluff. No emotional wording.

---

### 3. REDESIGN STRATEGY

Define the direction:

- Design style (choose one):
  - modern minimal
  - premium/luxury
  - bold/high-contrast
  - friendly/local community style

- Layout philosophy:
  - single-scroll homepage OR multi-section landing page
  - priority of information hierarchy

- Suggested structure:
  - Hero section
  - About business
  - Services / menu / products
  - Reviews / social proof
  - Location + map
  - Contact CTA

---

### 4. FEATURE UPGRADE LIST

Include:
- online booking / ordering (if relevant)
- contact form
- WhatsApp / call button
- menu/product gallery
- reviews integration
- SEO improvements
- mobile-first redesign

---

### 5. CONCRETE FIX LIST (ACTIONABLE)

5–10 direct changes:
- replace homepage hero
- fix navigation structure
- redesign typography system
- rebuild mobile layout
- compress images
- modernize color palette
- add CTA buttons
- improve spacing system

---

### 6. BUSINESS VALUE ANGLE

Explain in practical terms:
- improved customer trust
- higher conversion rate (calls, bookings, visits)
- better mobile usability (major traffic source)
- stronger local competition advantage

Keep factual, not persuasive marketing tone.

---

## 4. REDESIGN OUTPUT RULES

- No code required unless explicitly requested
- No full website cloning
- No copying competitor branding
- All designs must be original concepts
- No fake affiliation with business

---

## 5. OUTREACH POSITIONING RULES

Any future use of output must be framed as:

- “design improvement proposal”
- “website modernization concept”
- “freelance redesign pitch”

Never:
- claim ownership of their site
- imply official partnership
- deploy without permission

---

## 6. QUALITY CONTROL RULES

Reject results if:
- website is already modern
- business is large chain or corporate
- no official website exists
- only social media presence

---

## 7. OUTPUT STYLE

- direct
- structured
- technical
- no emojis
- no filler language
- no motivational tone
- no questions
- stop immediately after delivering content

---

## 8. END GOAL

Each run should produce:
- 1–2 real businesses
- clearly outdated websites
- a full redesign blueprint that can be shown to the business as a professional proposal
